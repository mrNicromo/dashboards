## Быстрый старт (macOS)

Если у вас **полная папка проекта** с файлами `LAUNCH.command`, `start.command` и каталогом `dashboard/`:

1. **Двойной клик** по `start.command` или `LAUNCH.command` в Finder (или в терминале: `./start.command` из корня проекта).
2. При первом запуске macOS может спросить разрешение; если скрипт не стартует — один раз выполните в терминале из этой папки:  
   `xattr -rd com.apple.quarantine . && chmod +x *.command`
3. Скрипт сам поднимет PHP в `portable/php` или, если PHP нет, **Node.js** (`dashboard/serve.mjs`). Порт по умолчанию: **9876**.
4. Токен: введите при запросе или задайте в терминале перед запуском `export AIRTABLE_PAT='pat_…'` (запишется в `dashboard/config.php`).

Подробности и переменные окружения: [README.md](../README.md) в корне проекта.

Дополнительно в `dashboard/`: [airtable-fields.md](airtable-fields.md) (какие поля Airtable читает код), [local-daily-use.md](local-daily-use.md) (локальный ежедневный сценарий), скрипт [open-dashboard.command](open-dashboard.command) — быстрый запуск сервера из папки `dashboard/`.

Ниже в разделе 2 описана **ручная** установка PHP и сервера — она нужна только если лаунчер не подходит.

## 1. Требования

| Компонент | Версия |
|-----------|--------|
| PHP | 8.1 или новее |
| PHP-расширения | `curl`, `json`, `mbstring`, `session` |
| Веб-сервер | Apache 2.4+ / Nginx / встроенный PHP-сервер |
| Интернет | Нужен для запросов в Airtable API и Google Sheets |
| Airtable PAT | Personal Access Token с правами `data.records:read`, `schema.bases:read` |

## 2. Установка на macOS

> **Рекомендуется:** сначала [быстрый старт](#быстрый-старт-macos) через `LAUNCH.command` / `start.command` в корне репозитория. Раздел ниже — для ручной настройки PHP, MAMP или Apache.

### Шаг 1 — Установить PHP

#### Вариант A: через Homebrew (рекомендуется)

```bash
# Установить Homebrew если ещё нет
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Установить PHP
brew install php

# Проверить версию
php -v
```

#### Вариант B: через MAMP (графический, проще для новичков)

1. Скачать [MAMP Free](https://www.mamp.info/en/downloads/) с официального сайта
2. Установить, запустить MAMP
3. В настройках MAMP → «Servers» → Document Root → указать папку с дашбордом
4. Запустить серверы кнопкой «Start»
5. Открыть `http://localhost:8888` в браузере

---

### Шаг 2 — Клонировать / скопировать файлы

```bash
# Перейти в нужную папку (например, домашний каталог)
cd ~/Sites

# Скопировать папку dashboard (или распаковать архив)
cp -r /путь/к/airtable/dashboard ./dashboard

cd dashboard
```

---

### Шаг 3 — Создать папку кэша

```bash
mkdir -p cache
chmod 775 cache
```

---

### Шаг 4 — Настроить конфиг

```bash
cp config.sample.php config.php
nano config.php
```

Заполнить поля (см. [раздел 4](#4-настройка-конфига)).

---

### Шаг 5 — Запустить встроенный PHP-сервер

```bash
cd ~/Sites/dashboard
php -S localhost:8080
```

Открыть в браузере: **http://localhost:8080**

> Для фоновой работы добавьте `&` в конце команды:
> `php -S localhost:8080 &`
> Остановить: `kill $(lsof -ti:8080)`

---

### Шаг 6 (опционально) — Apache через Homebrew

```bash
brew install httpd

# Редактировать конфиг Apache
nano /opt/homebrew/etc/httpd/httpd.conf
```

Найти строку `DocumentRoot` и изменить на путь к папке dashboard:

```apacheconf
DocumentRoot "/Users/ВАШ_ИМЯ/Sites/dashboard"
<Directory "/Users/ВАШ_ИМЯ/Sites/dashboard">
    AllowOverride All
    Require all granted
</Directory>
```

```bash
# Запустить Apache
brew services start httpd
```

## 3. Установка на Windows

### Вариант A: XAMPP (рекомендуется — самый простой способ)

1. Скачать [XAMPP](https://www.apachefriends.org/download.html) для Windows (PHP 8.x)
2. Установить в `C:\xampp` (путь без пробелов!)
3. Открыть **XAMPP Control Panel**
4. Запустить **Apache** (кнопка Start рядом с «Apache»)
5. Скопировать папку `dashboard` в `C:\xampp\htdocs\dashboard`

```
C:\xampp\htdocs\dashboard\
    index.php
    churn.php
    churn_fact.php
    manager.php
    config.php   ← создать из config.sample.php
    cache\       ← создать папку
    ...
```

6. Открыть в браузере: **http://localhost/dashboard**

---

### Вариант B: встроенный PHP-сервер (без Apache)

1. Скачать PHP для Windows: https://windows.php.net/download → **VS16 x64 Non Thread Safe**
2. Распаковать, например, в `C:\php`
3. Добавить `C:\php` в переменную PATH:
   - Win+R → `sysdm.cpl` → Дополнительно → Переменные среды
   - В «Системные переменные» найти `Path` → Изменить → Добавить `C:\php`
4. Открыть **PowerShell** или **Командную строку**:

```powershell
cd C:\путь\к\dashboard
php -S localhost:8080
```

5. Открыть браузер: **http://localhost:8080**

---

### Создать папку кэша (Windows)

В проводнике или через PowerShell:

```powershell
mkdir C:\путь\к\dashboard\cache
```

Убедитесь, что у папки `cache` есть права на запись (правой кнопкой → Свойства → Безопасность → полный доступ для текущего пользователя).

---

### Вариант C: через WSL (Windows Subsystem for Linux)

Если уже установлен WSL с Ubuntu:

```bash
sudo apt update
sudo apt install php php-curl php-mbstring php-json

cd /mnt/c/Users/ВАШ_ИМЯ/Desktop/airtable/airtable/dashboard
mkdir -p cache
php -S localhost:8080
```

## 4. Настройка конфига

Скопируйте `config.sample.php` → `config.php` и заполните:

```php
return [
    // Обязательно: Personal Access Token из Airtable
    // Получить: airtable.com → Account → Developer Hub → Personal access tokens
    // Права: data.records:read + schema.bases:read
    'airtable_pat' => 'patXXXXXXXXXXXXXX',

    // Обязательно: ID базы (appXXXXXXXXXXXXXX из URL Airtable)
    'airtable_base_id' => 'appEAS1rPKpevoIel',

    // Необязательно: если оставить пустым — система найдёт таблицы автоматически
    'airtable_dz_table_id'    => '',  // ID таблицы с долгами
    'airtable_dz_view_id'     => '',  // ID вьюхи «🔸Debt»
    'airtable_cs_table_id'    => '',  // ID таблицы клиентов
    'airtable_cs_view_id'     => '',  // ID вьюхи «CS ALL»
    'airtable_churn_table_id' => '',  // ID таблицы с Churn
    'airtable_churn_view_id'  => '',  // ID вьюхи «❤️ CHURN Prediction ☠️»
    'airtable_paid_view_id'   => '',  // ID вьюхи оплаченных
];
```

> **Безопасность:** `config.php` содержит секретный токен. Никогда не коммитьте его в Git.
> Добавьте в `.gitignore`:
> ```
> config.php
> cache/
> ```

### Альтернатива: переменные окружения

Вместо файла `config.php` можно задать переменные среды:

```bash
# macOS / Linux
export AIRTABLE_PAT="patXXXXXXXXXXXXXX"
export AIRTABLE_BASE_ID="appEAS1rPKpevoIel"
php -S localhost:8080

# Windows PowerShell
$env:AIRTABLE_PAT="patXXXXXXXXXXXXXX"
$env:AIRTABLE_BASE_ID="appEAS1rPKpevoIel"
php -S localhost:8080
```

Выкладка на хостинг (корень `dashboard/`, секреты, cron, Nginx): **[DEPLOY.md](DEPLOY.md)**. Пример переменных окружения — **`.env.example`** в корне репозитория.

## 5. Структура дашбордов

```
index.php         → Главная страница (сводка всех KPI)
ai_insights.php   → AI-аналитика (графики по кэшу + выводы AI: Gemini, при лимите — Groq; история в `cache/ai-insights-history.json`, ключи `DASHBOARD_GEMINI_API_KEY` и опционально `DASHBOARD_GROQ_API_KEY`)
churn.php         → Угроза Churn (кто сейчас в риске)
churn_fact.php    → Потери выручки (факт ухода и даунселла)
manager.php       → Дебиторская задолженность
```

### Навигация между дашбордами

Каждый дашборд имеет топбар со ссылками:

| Кнопка | Куда ведёт |
|--------|-----------|
| 🏠 | Главная (index.php) |
| ⚠ Угроза | churn.php |
| 💰 ДЗ | manager.php |

## 6. Главная — `index.php`

**Назначение:** быстрый обзор состояния всех трёх направлений без открытия отдельных дашбордов.

**Данные берутся из кэша** — живых запросов к Airtable нет. Кэш обновляется при открытии целевых страниц или через крон.

### Блоки главной

| Блок | Метрики |
|------|---------|
| 💰 Дебиторка | Общий долг, критический 91+ дней, кол-во клиентов, MRR |
| ⚠ Угроза Churn | MRR под риском, кол-во клиентов в риске, Prob=3 |
| 📉 Потери выручки | Churn YTD, DownSell YTD, % выполнения таргета |

**Если блок показывает нули** — кэш пустой. Откройте соответствующий дашборд и дождитесь загрузки, или запустите `cron_warmup.php`.

## 7. Угроза Churn — `churn.php`

**Назначение:** мониторинг клиентов с риском отказа от продукта. Источник данных — Airtable view «❤️ CHURN Prediction ☠️».

### KPI-карточки

| Карточка | Описание | Как читать |
|----------|---------|-----------|
| Клиентов в риске | Всего строк во вьюхе | Растёт = больше тревог |
| MRR под угрозой | Сумма MRR всех строк | В рублях |
| Prob=3 (Высокий) | Клиентов с макс. риском | Приоритет для работы |
| Прогноз потери | Prob3×1.0 + Prob2×0.6 + Prob1×0.3 | Взвешенный риск |

**Дельта-бейджи (↑↓)** — сравнение с предыдущей сессией, хранится в `localStorage` браузера.

### Панель фильтров

Фильтры находятся справа от KPI-карточек. Применяются одновременно (AND-логика).

| Фильтр | Значения |
|--------|---------|
| Сегмент | ENT, SME+, SME, SME-, SMB, SS |
| Вероятность | Prob 1 (низкий), Prob 2 (средний), Prob 3 (высокий) |
| Продукт | AnyQuery, AnyRecs, AnyImages, AnyReviews, Rees46, … |
| CSM | Конкретный менеджер из списка |

Кнопка «Сбросить» — убирает все фильтры.

### Пивот-таблицы

Четыре таба для агрегации данных:

| Таб | Группировка | Когда использовать |
|-----|------------|-------------------|
| По сегменту | ENT / SME / SMB | Общий разрез рисков |
| По CSM | Менеджер | Контроль нагрузки на CSM |
| По вертикали | Отрасль | Поиск проблемных вертикалей |
| По продукту | Продукт | Какой продукт теряем чаще |

### Таблица клиентов

- **Сортировка** — клик по заголовку колонки (повторный клик меняет направление)
- **Детали клиента** — клик по строке раскрывает подробности: продукты, вертикаль, дата последнего контакта, комментарий
- **Чекбокс «В работе»** — отметить клиента как обрабатываемого. Сохраняется в браузере между сессиями

### Кнопки топбара

| Кнопка | Действие |
|--------|---------|
| 📋 Сводка | Копирует текстовый дайджест (Клиентов, MRR, топ-5 Prob=3) в буфер обмена для Slack/почты |
| ⬇ CSV | Скачивает таблицу клиентов с текущими фильтрами |
| ⟳ | Принудительное обновление из Airtable (сбрасывает кэш) |
| 🌙/☀ | Тёмная / светлая тема |

**Автообновление** — каждые 5 минут страница сама подтягивает свежие данные в фоне.

## 8. Потери выручки — `churn_fact.php`

**Назначение:** анализ уже случившихся потерь — ушедшие клиенты (Churn) и снизившие MRR (DownSell). Источники: Airtable + Google Sheets CSV.

### KPI-карточки

| Карточка | Описание |
|----------|---------|
| Churn YTD | Потери от ушедших клиентов с 1 января |
| DownSell YTD | Потери от снижения MRR у действующих клиентов |
| Всего потерь | Churn + DownSell |
| ENT / SMB | Разбивка по корпоративному и малому сегменту |

### Waterfall-чарт (каскадный прогноз)

Горизонтальный график из 5 строк, показывает «движение» к таргету:

```
Таргет        ████████████████████████  8 200 000 ₽  — годовой план потерь
Churn         ████████                 −3 330 000 ₽  — уже случилось (красный)
DownSell      ██                          −93 000 ₽  — уже случилось (оранжевый)
Риск          ░░░░░░░░░░░░             −4 100 000 ₽  — Prob=3 из Airtable (пунктир)
Прогноз       ██████████████████████    7 500 000 ₽  — итоговый прогноз
```

- **Зелёный буфер** справа — запас до таргета (хорошо, если есть)
- **«Риск»** импортируется автоматически из дашборда Угрозы Churn
- Наведение на строку — тултип с точными суммами

### Месячный bar-чарт

Столбчатый график потерь по месяцам.

| Элемент | Описание |
|---------|---------|
| Синий столбец | Churn за месяц |
| Оранжевый столбец | DownSell за месяц |
| Цифры над столбцом | Суммарные потери месяца |
| **Клик по столбцу** | Фильтрует таблицы ниже только по этому месяцу |
| Повторный клик | Сбрасывает фильтр месяца |

> При активном фильтре: выбранный столбец выделяется, остальные затемняются. Это позволяет детально разобрать конкретный месяц.

### Табы таблиц

**Таб «Churn» — ушедшие клиенты:**

| Колонка | Описание |
|---------|---------|
| Клиент | Название аккаунта |
| Сегмент | ENT / SMB |
| Месяц | Год-месяц ухода (напр. 2025-03) |
| MRR | Потерянный MRR в рублях |
| Причина | Причина ухода («Не указана» если пусто) |

**Таб «DownSell» — снизившие MRR:**

| Колонка | Описание |
|---------|---------|
| Клиент | Название аккаунта |
| Сегмент | ENT / SMB |
| Месяц | Дата изменения |
| Изменение MRR | Дельта (отрицательная сумма) |

### Кнопки топбара

| Кнопка | Действие |
|--------|---------|
| ⟳ | Обновить данные (Airtable + Google Sheets) |
| 🌙/☀ | Тёмная / светлая тема |
| ⚠ Угроза | Перейти на churn.php |
| 💰 ДЗ | Перейти на manager.php |
| 🏠 | Главная |

**Кнопка «↓ CSV»** в заголовке каждой таблицы — скачать текущий (отфильтрованный) список в `.csv`.

## 9. Дебиторская задолженность — `manager.php`

**Назначение:** контроль просроченных платежей клиентов. Источник — Airtable таблица ДЗ.

### Группы просрочки

| Группа | Срок | Цвет |
|--------|------|------|
| Текущие | 0–30 дней | Зелёный |
| 31–60 | 1–2 месяца | Жёлтый |
| 61–90 | 2–3 месяца | Оранжевый |
| 91+ | Более 3 месяцев | Красный |

Строки 91+ выделены красным цветом в таблице.

### Основная таблица

- Сортировка по любой колонке (клик на заголовок)
- Фильтр по CSM (менеджеру)
- Клик по строке — детальная информация о клиенте

### Блок «Угроза Churn» (сайдбар)

Показывает топ-5 клиентов из кэша дашборда Угрозы — без живого запроса в Airtable. Обновляется при открытии `churn.php`.

### График оплат по неделям

Гистограмма: сколько поступало оплат за последние недели. Помогает отслеживать динамику инкассации.

## 10. Автообновление и кэш

### Как работает кэш

```
Возраст кэша    Поведение
0–15 минут   →  Данные свежие, новый запрос не делается
15–30 минут  →  Отдаётся кэш, ПАРАЛЛЕЛЬНО запускается фоновое обновление
30+ минут    →  Принудительный запрос, страница ждёт ответа
```

Кнопка **⟳** всегда делает принудительное обновление независимо от возраста кэша.

### Где хранится кэш

Файлы JSON в папке `cache/`:

| Файл | Данные |
|------|--------|
| `cache/churn-report.json` | Угроза Churn |
| `cache/churn-fact-report.json` | Потери выручки |
| `cache/dz-data-default.json` | Дебиторка |

### Автообновление в браузере

Каждые **5 минут** JS автоматически запрашивает API и обновляет данные без перезагрузки страницы. Таймер сбрасывается при ручном обновлении (⟳).

## 11. Крон-прогрев кэша

Скрипт `cron_warmup.php` заранее обновляет кэш по расписанию — чтобы первый посетитель утром не ждал загрузки.

### Запуск вручную

```bash
# Прогреть все три отчёта
php cron_warmup.php

# Только один отчёт
php cron_warmup.php --only=churn
php cron_warmup.php --only=fact
php cron_warmup.php --only=manager

# Принудительно (сбросить кэш перед прогревом)
php cron_warmup.php --force
```

### macOS / Linux — настройка cron

```bash
# Открыть crontab
crontab -e

# Добавить строку: каждый день в 22:00 МСК (19:00 UTC)
0 19 * * * php /полный/путь/к/dashboard/cron_warmup.php >> /var/log/cron_warmup.log 2>&1
```

> Используйте `which php` чтобы найти путь к PHP, например `/opt/homebrew/bin/php`.

```bash
# Пример с полным путём к PHP
0 19 * * * /opt/homebrew/bin/php /Users/ИМЯ/Sites/dashboard/cron_warmup.php >> ~/cron_warmup.log 2>&1
```

### Windows — настройка Планировщика задач

1. Открыть «Планировщик задач» (Task Scheduler)
2. «Создать простую задачу» → имя: `AnyQuery Cache Warmup`
3. Триггер: ежедневно, время 22:00

## 12. Отладка и устранение проблем

### Страница отладки данных

Откройте в браузере:
```
http://localhost:8080/churn_fact_debug.php
```

Показывает полный дамп: сколько строк прочитано из Airtable, какие поля найдены, сколько строк отфильтровано и почему.

### Частые проблемы

#### Все данные нулевые / не загружаются

1. Проверьте `config.php` — заполнен ли `airtable_pat`
2. Проверьте папку `cache/` — она должна существовать и быть доступна для записи
3. Откройте в браузере `churn_fact_api.php` напрямую — увидите JSON или ошибку

#### Все Churn попали в март (неправильный месяц)

Airtable-вьюха использует специфическое название поля даты. Проверьте `churn_fact_debug.php` — секция `_dateSamples`. Поле должно называться одним из:
- `Дата ухода AQ`, `Дата ухода AC`, `Дата ухода RECS`, `Дата ухода Recs`
- `Дата ухода AnyImages`, `Дата ухода AnyReviews`, `Дата ухода Rees46`
- `Дата ухода`, `Date`, `Дата`

#### DownSell показывает 0 строк

DownSell берётся из Google Sheets CSV. Проверьте:
- Колонка `type` должна содержать слово `down` (регистр не важен) или быть пустой
- Колонка `change` должна быть отрицательным числом
- URL к Google Sheets должен быть доступен без авторизации (лист открыт для всех)

#### ENT/SMB прогноз показывает «Риск 0 ₽» после авто-обновления

Убедитесь, что в `churn_fact_api.php` передаются `$prob3riskEnt` и `$prob3riskSmb` в `ChurnFactReport::fetchReport()`. Это должно быть исправлено в текущей версии.

#### Ошибка «CSRF token mismatch» (403)

- Из интерфейса Churn: запросы идут с заголовком `X-CSRF-Token` из `<meta name="csrf-token">`. При нескольких вкладках обновите страницу (F5).
- Прямой вызов в браузере или с Railway/cron **без сессии**: задайте секрет и передайте его одним из способов:
  - Переменная окружения **`DASHBOARD_API_SECRET`** (или поле **`api_secret`** в `config.php`), длинная случайная строка.
  - Запрос: заголовок `Authorization: Bearer <секрет>` или `X-Api-Key: <секрет>`, либо параметр **`api_secret`** (GET/POST). Пример: `churn_api.php?api_secret=...` (секрет в URL может попасть в логи — для продакшена лучше Bearer).

Пока секрет **не задан**, для API по-прежнему нужен только CSRF из залогиненной страницы.

#### PHP Warning: Cannot write cache file

```bash
# macOS / Linux
chmod 775 cache/
chown $(whoami) cache/

# Windows — дать права через проводник:
# Правой кнопкой на папке cache → Свойства → Безопасность → Изменить → полный доступ
```

### Проверка PHP-расширений

```bash
php -m | grep -E "curl|json|mbstring"
```

Все три должны присутствовать в выводе. Если нет — установить:

```bash
# Ubuntu / Debian / WSL
sudo apt install php-curl php-mbstring php-json

# macOS Homebrew
brew install php   # все расширения идут в комплекте
```

## Быстрый старт — одна команда (TL;DR)

### macOS

```bash
# Сделать скрипт исполняемым (один раз)
chmod +x start.sh

# Запустить — установит PHP, создаст кэш, настроит конфиг, откроет браузер
./start.sh
```

Скрипт сам установит Homebrew и PHP если их нет, спросит Airtable PAT и откроет браузер.

### Windows (PowerShell)

```powershell
# Разрешить выполнение скриптов (один раз, от имени пользователя)
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

# Запустить
.\start.ps1
```

Скрипт найдёт PHP (XAMPP / PATH) или автоматически скачает PHP 8.3, настроит конфиг, откроет браузер.
