# Локальный дашборд для ежедневной работы

Все данные и архивы хранятся локально на этом ПК:

- `dashboard/snapshots/*.html` — сохраненные версии HTML-отчетов.
- `dashboard/snapshots/manifest.json` — индекс архива версий.
- `dashboard/config.php` — токен (не в git).

## 1) Быстрый старт (одним файлом)

1. Двойной клик по файлу `Запуск дашборда.command` (в корне проекта).

Что делает файл:
- если локальный сервер еще не запущен, запускает его;
- если уже запущен, просто использует его;
- если порт занят другим приложением, автоматически выберет свободный;
- если не хватает служебных файлов (архив, манифест), создаст их;
- открывает дашборд в браузере.

**PHP не обязателен:** если `php` не установлен, используется **Node.js** (`serve.mjs`). Достаточно установленного Node (как у вас в системе).

Альтернатива через терминал:

```bash
cd /Users/d.arkhangelskiy/Desktop/airtable/dashboard
bash run-local.sh
# или без PHP:
node serve.mjs
```

## 2) Ежедневный сценарий руководителя

- Нажмите `Обновить данные` для актуализации из Airtable.
- Нажмите `Скачать HTML`:
  - создается новая архивная версия в `dashboard/snapshots/`;
  - файл сразу открывается/скачивается;
  - в блоке `История и архив версий` появляется новая строка.
- При необходимости включите авто-обновление кнопкой `Авто`.
- Кнопка `LIVE` — периодическое обновление **выжимки** (без полной перерисовки таблицы); при сбое сети появится предупреждение и кнопка «Повторить» (то же действие, что «Обновить данные»). Ошибка «Обновить данные» или «Скачать HTML» тоже отражается в выжимке и баннере.

### Фильтры, ссылка и пресеты

- Над полями фильтров: **чипы** (просрочка, сброс), кнопка **«Ссылка с фильтрами»** — копирует URL с параметром `dzf` (срез можно вставить в закладку или отправить коллеге). Пока включена опция в меню ☰ «Подставлять фильтры в адрес», при смене фильтров адрес обновляется сам.
- В меню ☰ → **«Сохранить текущий срез»** — именованный пресет (до 8 шт.); загрузка — выпадающий список **«Пресет»** над фильтрами.
- Блок **«Исключения (срез и выжимка)»**: первый подблок убирает строки везде (включая таблицу); второй — **«Не в выжимку и отчётах»**: строки остаются в таблице, но не попадают в KPI, выжимку, сводку/аналитику и отчёт «под ключ» (параметры `dzf` тоже сохраняются).
- Строка **«К разделу»** вверху страницы — быстрый переход и переключение вкладки (Сводка / Аналитика / Счета и т.д.).

### Меню ☰ (оформление и пороги)

- **Тема** — тёмная / светлая / как в системе.
- **Таблица счетов** — комфорт или компактная сетка.
- **Скрыть суммы** — маскирует суммы в выжимке, срезах и CSV.
- **Пороги в выжимке** — при превышении доли просрочки (%) или суммы просрочки (₽) показывается заметное предупреждение.

### Таблица счетов и Airtable

- У строки с привязкой к записи: **Enter** или **клик** по строке (не по кнопкам) — открыть эту запись в Airtable в новой вкладке.

### Обновление по расписанию (сервер)

- Пример скрипта для cron: `dashboard/scripts/refresh-dashboard-cron.example.sh` (переменная `DZ_REFRESH_URL` — полный URL к `api.php?action=refresh`).

## 2а) Дымовой тест UI (опционально)

Нужен **Node.js** (как для `serve.mjs`), затем из корня проекта:

```bash
npm install
npx playwright install chromium
npm run test:e2e
```

Playwright поднимет `node dashboard/serve.mjs` на порту `9876` и проверит, что `index.php` отдаёт страницу дашборда (тот же сценарий, что «без PHP» в быстром старте). Уже запущенный сервер на этом порту будет переиспользован.

## 3) Восстановление/скачивание старых версий

В блоке `История и архив версий` есть ссылки `Скачать HTML` на каждую сохраненную версию.

## 4) Полная локальность

- Никаких внешних серверов для дашборда не требуется.
- Нужен только локальный PHP-сервер и доступ к Airtable API.

## 5) Автостарт локального сервера (macOS, опционально)

В папке лежит шаблон `com.airtable.dz-dashboard.plist.example`.

1. Скопируйте файл в `~/Library/LaunchAgents/com.airtable.dz-dashboard.plist`.
2. Загрузите:

```bash
launchctl load ~/Library/LaunchAgents/com.airtable.dz-dashboard.plist
```

3. После входа в систему сервер будет запускаться автоматически.
