# Какие поля Airtable использует дашборд

Дашборд читает данные через Airtable REST API v0 из **таблицы с ДЗ** (Debt), полный список строк из **«CS ALL»** для вкладки «Клиенты» и полный список из **«❤️ CHURN Prediction ☠️»**, который **присоединяется к строкам CS ALL** по названию клиента. Остальные связанные таблицы **не запрашиваются**; в Debt по-прежнему используются только поля, пришедшие в записи (в т.ч. lookup).

---

## База и таблица

| Параметр | Значение |
|----------|----------|
| База | задаётся в `config.php` → `airtable_base_id` (по умолчанию `appEAS1rPKpevoIel`) |
| Таблица (имя в Airtable) | **🔸Debt (15,30,60,90)(Демидова) copy** |
| Table ID | Если в `config.php` задан `airtable_dz_table_id` (или переменная `AIRTABLE_DZ_TABLE_ID`) — используется он. Иначе ID ищется по **точному совпадению имени** через Meta API (`GET …/meta/bases/{baseId}/tables`). У токена должен быть доступ к метаданным; иначе укажите `tbl…` вручную (страница API для вашей базы на airtable.com). |

В JSON-ответе дашборда в `meta.table` попадает фактический Table ID, в `meta.tableName` — ожидаемое имя таблицы.

### Таблица «CS ALL» (клиенты)

| Параметр | Значение |
|----------|----------|
| Назначение | Справочник клиентов для вкладки **«Клиенты»** в дашборде |
| Имя в Airtable | **CS ALL** (точное совпадение) |
| Table ID | `airtable_cs_table_id` в `config.php` или `AIRTABLE_CS_TABLE_ID`; иначе поиск по имени через Meta API (как у Debt) |
| Запрос | Все строки таблицы, **без** `filterByFormula` |
| Сшивка с ДЗ | Для каждой строки CS ALL в колонке **«ДЗ в срезе»** подставляется сумма по тем же отфильтрованным счетам Debt, у которых **«ЮЛ клиента»** совпадает с **отображаемым названием клиента** (поля в порядке приоритета: `ЮЛ клиента`, `ЮЛ`, `Account`, `Клиент`, … — см. `CS_ALL_LABEL_KEYS` в `DzReport.php`). Сравнение также пробует вариант **без учёта регистра**. |
| Ошибки | Если таблицу не найти или запрос не удался, `csAllClients` будет пустым, текст ошибки — в `meta.csAll.error`; дашборд по Debt продолжит работать. |

### Таблица «❤️ CHURN Prediction ☠️»

| Параметр | Значение |
|----------|----------|
| Назначение | Данные прогноза оттока в колонке **CHURN** на вкладке «Клиенты» (рядом с CS ALL) |
| Имя в Airtable | **❤️ CHURN Prediction ☠️** (точное совпадение) |
| Table ID | `airtable_churn_table_id` в `config.php` или `AIRTABLE_CHURN_TABLE_ID`; иначе поиск по имени через Meta API |
| Запрос | Все строки таблицы, **без** `filterByFormula` |
| Сшивка с клиентами | Для каждой строки CHURN определяется **отображаемое имя клиента** тем же списком полей, что и для CS ALL (`CS_ALL_LABEL_KEYS` в `DzReport.php`). По **нормализованному** названию (`trim` + нижний регистр, в PHP — `mb_strtolower`) строится карта → в каждую строку `csAllClients` добавляется поле `churnDetails` (до 8 полей записи, кроме ключа названия; несколько строк с одним клиентом склеиваются через ` · ‖ `). |
| Ошибки | При сбое CHURN колонка пустая; текст — в `meta.churn.error`; Debt и CS ALL продолжают работать. |

---

## Фильтр записей (какие строки попадают в отчёт)

В API используется **filterByFormula**:

```text
OR({Статус оплаты}='Не оплачен',{Статус оплаты}='Оплачен частично')
```

То есть в дашборд попадают только счета со статусом **«Не оплачен»** или **«Оплачен частично»**. Строки «Оплачен» в выборку **не входят**.

## Поля таблицы с ДЗ, которые читает код

| Поле в Airtable | Зачем используется |
|-----------------|---------------------|
| **Статус оплаты** | Фильтр записей, колонка, фильтр UI, логика просрочки |
| **Фактическая задолженность** | Основной источник суммы ДЗ (formula), см. порядок ниже |
| **Сумма остатка** | Остаток при частичной оплате; резерв, если формула пустая/0 |
| **Сумма счета** | Резервная сумма, если нет ни формулы, ни остатка |
| **Срок оплаты** | Просрочка, даты, фильтры |
| **Кол-во дней просрочки** | Сортировка, просрочка (formula) |
| **Группа просрочки** | Возрастные корзины 0–30 / 31–60 / 61–90 / 90+ (formula) |
| **ЮЛ клиента** | Группировки, топ контрагентов, таблица |
| **Номер счета** | Таблица, поиск; **период счёта** (месяц YYYY-MM) выводится из номера (шаблоны даты, квартал, рус. месяц) или из **Дата счета** |
| **ИД счета** | Таблица (идентификатор) |
| **Наша компания** | Срез по юрлицу выручки |
| **Направление** | Продуктовый срез |
| **Аккаунт менеджер** | Срез по менеджерам (обычно lookup из связанных записей) |
| **Следующий шаг_ Статус_** | Таблица (rich text) |
| **Срок по шагу** | Таблица |
| **Комментарий по ДЗ** | Таблица, поиск |
| **Дата счета** | Таблица |
| **Статус отгрузки** | Таблица |
| **Статус отправки** | Таблица |
| **Иск** | Таблица |

Остальные поля этой таблицы дашборд **не использует** (но они могут быть в Airtable для других процессов).

## Как считается сумма задолженности по строке (важно для «0 ₽»)

Иногда API отдаёт суммы **числом**, иногда формула приходит **строкой** (локальный формат). Код:

1. Парсит значение как число, в т.ч. из строк с пробелами/запятой.
2. **Если статус «Оплачен частично»:** сначала берётся **Сумма остатка** (если есть ненулевая), затем **Фактическая задолженность**, затем **Сумма счета**.
3. **Иначе:** сначала **Фактическая задолженность**, затем **Сумма остатка**, затем **Сумма счета**.

Если после этого сумма всё ещё 0 — проверьте в Airtable, что для этой строки заполнено хотя бы одно из трёх полей и что статус попадает под фильтр отчёта.

## Где это реализовано в коде

| Файл | Назначение |
|------|------------|
| `dashboard/lib/DzReport.php` | Сбор payload для PHP (`index.php`, `api.php`), в т.ч. CS ALL и CHURN |
| `dashboard/serve.mjs` | Тот же сбор при запуске через Node |
| `dashboard/lib/ReportHtml.php` | HTML-снимок: счета + таблица CS ALL с CHURN |
| `dashboard/assets/dashboard.js` | Краткая выжимка, **«Отчёт под ключ»** (превью + графики Chart.js с CDN), выгрузки HTML/XML/CSV и скачивание `dashboard.css`; **выбор таблицы** в шапке (список из Meta API) |
| `api.php?action=tables` | Список таблиц базы и `defaultDebtTableId` (нужен доступ токена к метаданным) |
| `api.php?action=refresh&tableId=tbl…` | Загрузка **всех записей** выбранной таблицы без фильтра ДЗ (`meta.dataMode: generic`); пустой `tableId` — режим дебиторки как раньше |
| `build_dz_report.py` | Статический HTML-отчёт в корне проекта |

Логика суммы синхронизирована между этими тремя местами.
